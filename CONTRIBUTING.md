# Contributing to the AIDLC Handbook

Thank you for your interest in evolving the AI-Powered Development Life Cycle! Because the field of Agentic AI moves so fast, this handbook is designed to be a **living document**.

## 🛠 How to Contribute

We welcome contributions in the following areas:
1. **New Agentic Workflows:** If you've discovered a more efficient way to orchestrate AI for coding, QA, or ideation.
2. **Prompt Optimization:** Improved "Golden Prompts" for the Appendix that yield better results across different models (Claude, GPT-4, etc.).
3. **Case Studies:** Real-world examples of a project that transitioned from a traditional SDLC to an AIDLC.
4. **Tooling Updates:** Updated recommendations in the Appendix as new AI IDEs or Agents emerge.

## 📝 Contribution Process

1. **Fork the Repository:** Create your own copy of the project.
2. **Create a Feature Branch:** `git checkout -b feature/your-improvement`.
3. **Implement Your Change:** Edit the Markdown files. Ensure you follow the "Contrast $\rightarrow$ Workflow $\rightarrow$ Lab" structure used in the modules.
4. **Open a Pull Request:** Describe what you changed and *why* it improves the AI-powered lifecycle.

## 📐 Style Guidelines
- **Keep it Actionable:** Avoid vague theory. Provide prompts, command examples, and concrete steps.
- **Stay Model-Agnostic:** Write guidelines that work across the major LLM providers.
- **Maintain the Comparison:** Always highlight the difference between the "Traditional" and "AI-Powered" way of doing things.

**Let's build the future of software together!**
